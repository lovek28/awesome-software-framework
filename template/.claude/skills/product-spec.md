# Skill: Product spec (stage: product_spec)

**When:** Current stage is product_spec. Apply when writing or updating spec/product/spec.md.

**What to do:**
- Fill Problem (who and what problem), Target Users, Core Features (high-level list), and at least one Functional Requirement ("The system shall…" or "Users must be able to…").
- Add Non-Functional Requirements and Constraints if relevant. Keep structure product-agnostic; content project-specific.
- Before marking stage done, ensure quality gate: Problem, Target Users, Core Features, and ≥1 Functional Requirement present. If not, ask the user to fill gaps.

**Constraints:** Do not write implementation; only the product spec. Update spec/index.md after significant changes.
